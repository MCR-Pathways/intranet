# Resources ¬ß2.1b ‚Äî flag the existing auto-detected grids

**Date:** 2026-07-01
**Status:** Design approved (2026-07-01), pending spec review. Follows ¬ß2.1 (editor-controlled resource cards, #380). Independent of ¬ß4 (reading rail) and the parked accessibility follow-ons (k32‚Äìk35).

## Problem

¬ß2.1 added a per-link `displayAsCard` flag but kept auto-detection as the default: a run of 4+ consecutive resource cells still renders as a card grid whether or not the links carry the flag. So every grid that exists today is a grid **by auto-detection, not by flag**. The moment an editor deletes a link and drops a run below 4, auto-detection stops firing and the remaining links revert to inline ‚Äî a grid silently falls apart mid-edit.

The scope is concrete. Three published native articles hold grids today:

- `people-services` ‚Äî one run of 4.
- `group-work` ‚Äî runs of 4, 17, 16, 17, 7, 12.
- `pc-support` ‚Äî runs of 12, 8, 6.

That is 103 standalone-links across the three, none flagged, no files in any run (every grid is a pure link-grid). The runs currently at exactly 4 ‚Äî `people-services` and the first `group-work` run ‚Äî revert on a single deletion; the larger runs have headroom but shrink the same way over enough edits.

## Design

Keep auto-detection. ¬ß2.1b is purely additive: set `displayAsCard: true` on the links already inside 4+ runs so those grids survive a sub-threshold edit. Auto-grid-at-4+ still fires for new content ‚Äî the zero-effort default ¬ß2.1 shipped is untouched.

### The load-bearing invariant

Flag links **only in runs of `MIN_GRID_RUN` (4) or more**. A run of 3 renders inline today; flagging it would promote 3 inline links into a 3-card grid, a visible change. Gating on `>= 4` means the migration flags exactly what is already a grid, so the grid structure `groupResourceGrids` produces is unchanged for every article: the same runs become grids, with the same boundaries and the same membership. The only difference in the grouped tree is the `displayAsCard` flag now present on the member links. Put precisely: strip `displayAsCard` from every node and `groupResourceGrids(after)` deep-equals `groupResourceGrids(before)`. Same render today, robust tomorrow. This is asserted in a test.

### Two units

**Pure transform.** `flagAutoGridCards(value: Value): { value: Value; flaggedCount: number }`, added to `src/lib/resource-grid.ts` beside the detection logic it reuses (`MIN_GRID_RUN`, `isResourceCell`, `standaloneLink` ‚Äî all already there, so the threshold stays defined once). It walks the top-level array, finds each maximal run of 4+ consecutive resource cells, and for every **standalone-link** cell in such a run sets `displayAsCard: true` on the inner `a` node. It builds a new value and never mutates the input. It is idempotent: a link already flagged stays flagged and is not re-counted, so `flaggedCount` is the number of links newly flagged this pass and a second run reports 0. File voids in a run are left untouched (files carrying the flag is a ¬ß2.1 v2 item; there are none in any grid today regardless).

**One-off script.** `scripts/flag-auto-grid-cards.ts`, mirroring `scripts/flatten-toggles.ts`: an env-sourced service-role Supabase client, fetch all `content_type = "native"` articles, run `flagAutoGridCards` on each, `--dry-run` prints `slug [status]: N flagged` per article and writes nothing, apply writes `content_json` back with the `Database["public"]["Tables"]["resource_articles"]["Update"]["content_json"]` cast. Bulk (one pass over all native articles), idempotent, re-runnable. Deterministic with no per-page decisions, unlike `flatten-toggles.ts`, which needed per-page exclude/heading arguments.

## Index-safety ‚Äî no reindex

`displayAsCard` is presentational. The Algolia serialisation path (`createNativeStaticEditor` ‚Üí `serialiseContentToHtml`) renders links inline and never groups, so it ignores the flag: a flagged link serialises identically to an unflagged one. The indexed HTML and the section records are byte-identical before and after the migration, so no reindex is needed. This is the same index-safety contract ¬ß2 and ¬ß2.1 established; it will be re-confirmed against `serialiseContentToHtml` during implementation.

## Scope

All native articles, published and draft ‚Äî not only the three with grids today. A draft can hold a 4+ run too, and the flag is inert until a grid actually shrinks, so flagging every current grid is consistent and future-proof rather than a special-case pass over three slugs.

## Edge cases and resolutions

- **Idempotent re-run** ‚Äî an already-flagged link stays flagged and is not counted again; a second run reports 0 flagged and writes nothing.
- **Sub-4 run** ‚Äî never flagged. This is the invariant above: a run of 3 renders inline today and must keep rendering inline.
- **File in a 4+ run** ‚Äî left unflagged (files-as-cards is v2). None exist today, but the transform handles a mixed run correctly if one appears: the links flag, the file is untouched.
- **Lone flagged link outside any run** ‚Äî untouched. A link an editor already flagged as a lone card (not part of a 4+ run) is not in scope and keeps its flag.
- **Run of exactly 4** ‚Äî the acute case (`people-services`, first `group-work` run). Flagging it means a single deletion drops the run to 3 flagged links that stay cards instead of reverting to inline.
- **`content_json` missing or not an array** ‚Äî skipped, matching `flatten-toggles.ts`.

## Files

- `src/lib/resource-grid.ts` ‚Äî add `flagAutoGridCards`; export it. Reuses the existing private `MIN_GRID_RUN` and the exported `isResourceCell` / `standaloneLink`.
- `src/lib/resource-grid.test.ts` ‚Äî unit tests for the transform (see below).
- `scripts/flag-auto-grid-cards.ts` ‚Äî the one-off bulk migration script.

## Tests

Unit tests on `flagAutoGridCards` (the risky part; the script is I/O glue):

- A run of 4+ standalone links ‚Üí every link flagged; `flaggedCount` equals the run length.
- A run of 3 standalone links ‚Üí nothing flagged (`flaggedCount` 0); the value is unchanged.
- A run containing a file void among links ‚Üí the links flag, the file node is untouched.
- An already-flagged link inside a run ‚Üí stays flagged, not double-counted; a re-run reports 0.
- A lone standalone link outside any 4+ run ‚Üí untouched.
- Two runs split by a non-cell block (e.g. a heading) ‚Üí each run handled independently.
- Equivalence: for a value with a 4+ run, the grid structure is unchanged by the migration ‚Äî with `displayAsCard` stripped from every node, `groupResourceGrids(flagAutoGridCards(v).value)` deep-equals `groupResourceGrids(v)`. (Without stripping they differ only by the flag, which is the point.)

## Out of scope

- **Retiring auto-detection.** Considered and rejected: it conflicts with ¬ß2.1's decision to keep auto-grid-at-4+ as the zero-effort default, and it would force editors to tick each link on new content. ¬ß2.1b keeps auto-detection; the flag makes existing grids robust, it does not replace the mechanism.
- **Opt-out** (`displayAsCard: false` to force a link inline within a 4+ run). A ¬ß2.1 v2 item, unaffected here.
- **Files as cards.** The flag stays links-only in v1; file voids are card-by-auto. Out of scope for this migration.
- **Flag-on-save (lazy migration).** Rejected: it mutates content invisibly on every save, only fixes articles that happen to get re-saved, and adds cost to the save hot path. The explicit one-off script is cleaner and covers every article in one pass.
