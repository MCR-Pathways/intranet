Design DNA — What We're Doing Well

Every item below is explicitly preserved. None of the 4 proposed changes affect any of these. This is our non-negotiable baseline.

Layout & Structure

✓
Facebook-style centred feed column — Content is constrained to 590px and horizontally centred. Creates a focused reading experience regardless of viewport width. Grey space visible on both sides. intranet/page.tsx → max-w-[590px] mx-auto
✓
Grey page background + white card surfaces — #F2F4F7 page background with white cards using shadow-sm (posts) and shadow-md (tables). Cards are the primary content surface. Industry standard (GitHub, Notion, Linear). globals.css → --background: #F2F4F7, --card: white
✓
Collapsible sidebar with persistence — Expanded (w-64) and collapsed (w-16) states. Collapse state persists in localStorage and syncs across tabs via storage event. Tooltip fallbacks in collapsed mode. Smooth 200ms transition. sidebar.tsx, app-layout.tsx → useSyncExternalStore
✓
Responsive container with padding scale — Content capped at max-w-7xl. Padding scales by breakpoint: p-4 → md:p-6 → lg:p-8. Ensures breathing room on large screens. app-layout.tsx → max-w-7xl p-4 md:p-6 lg:p-8

Navigation & Headers

✓
Sticky header with z-50 — Header stays above content when scrolling. Hamburger, logo, bell, and avatar aligned. Right-aligned user controls with avatar dropdown showing name, email, and job title. header.tsx → sticky top-0 z-50
✓
Breadcrumbs with / separator — Text / separators (not chevron icons). Links have hover:underline underline-offset-4 for click affordance. Applied consistently across all 24 pages. page-header.tsx → select-none aria-hidden
✓
Page title + subtitle hierarchy — h1 at text-3xl font-bold, subtitle in text-muted-foreground. Right-aligned action buttons. Consistent across every page. page-header.tsx → PageHeader component
✓
Dark navy active pill in sidebar — Active item: bg-primary text-primary-foreground (#213350 + white). Child items indent with left border. Section labels uppercase with letter-spacing. sidebar.tsx → isItemActive()

Social Feed

✓
Pill-shaped composer trigger — Avatar + rounded-full input reading "Share something with the team..." + action bar below with Photo/Document/Poll. Matches Facebook/LinkedIn convention. Low-friction composition. post-composer.tsx → rounded-full bg-muted px-4 py-2.5
✓
Optimistic reactions — Reaction counts update immediately on click before server round-trip. Uses pending state to avoid double-taps. Feels instant. post-card.tsx → useTransition + optimistic state
✓
Pinned/roundup visual distinction — Pinned posts get coloured border + "Pinned" badge with pin icon. Weekly roundups get distinct background. Both stand out from regular posts. post-card.tsx → conditional border and bg
✓
Post action bar with border-top — Like/Comment/Share actions separated from content by a 1px border-top. Consistent spacing and icon alignment. Matches social platform patterns. post-card.tsx → border-top: 1px solid

Data Tables

✓
Card wrapper with shadow-md — Tables sit inside bg-card shadow-md rounded-xl overflow-clip. More visual weight than post cards. Overflow-clip preserves sticky headers without breaking child scroll containers. data-table.tsx → card wrapper
✓
Dedicated table header token #E4E7EC — Not bg-muted (#F0F2F5) which blends into the page. Purpose-built contrast between page, card, and header surfaces. globals.css → --table-header: #E4E7EC
✓
Search toolbar inside card — Search input with magnifying glass icon + filter dropdowns. All within the card surface, not floating above. Toolbar is part of the table, not separate. data-table.tsx → toolbar section
✓
"Showing X of Y results" footer — Footer with border-top inside the card. Shows total count when external filtering is active. Pagination controls on the right. data-table.tsx → totalCount prop
✓
Sticky headers with border-separate — border-separate border-spacing-0 prevents visual glitches when combined with position: sticky. Borders on th cells, not tr rows. data-table.tsx → border-separate
✓
Sortable column headers with font-semibold — Ghost Button gets font-semibold override to match non-sortable headers. Sort indicator arrows visible on hover/active. DataTableColumnHeader → font-semibold

Component Patterns

✓
Line + pill tab variants — Line tabs for top-level (border-bottom, after:: pseudo-element underline). Pill tabs for nested sections (bg-muted background). Both get hover:text-foreground. Propagated via data-variant attribute. tabs.tsx → cva with default + line variants
✓
Semantic badge variants — success (green), warning (amber), destructive (red), muted, outline. Rounded-full pills. Status colours use dedicated tokens. Compact text-xs sizing. badge.tsx → cva with 7 variants
✓
Location nudge banner — Warm amber (#FEF9E7 bg, #FDE68A border). Full-width within content area, above the centred feed column. Location pills with orange border. Dismiss button on right. daily-banner.tsx → amber colour palette
✓
Primary navy buttons — #213350 background, white text. Used for primary CTAs (New Category, Add User, Save). Rounded-lg (8px). Clear, professional, consistent. globals.css → --primary: var(--mcr-dark-blue)

Design Tokens & Theming

✓
Decoupled accent/secondary tokens — --accent (#E2E8F0) for hover/focus states. --secondary (#E8ECF0) for badges/surfaces. Independent tuning in both light and dark mode. globals.css → separate hex values in :root and .dark
✓
Complete dark mode token parity — Every semantic token has a dark mode equivalent. Proper contrast ratios maintained. Dark mode is not an afterthought. globals.css → .dark block
✓
Status colour tokens — --status-active (green), --status-pending (amber), --status-inactive (red). Used by badges, not hardcoded per component. globals.css → status token block
✓
bg-card for inputs, not bg-background — Form controls use bg-card (white) for visibility on grey page background. On white dialogs, the border provides sufficient contrast. CLAUDE.md lesson → bg-card for inputs

Accessibility & Motion

✓
Focus-visible with 2px offset — Global focus ring: 2px outline, 2px offset. Applied via :focus-visible (not :focus). Keyboard navigators see clear focus indicators; mouse users don't. globals.css → :focus-visible
✓
Reduced motion support — All animations respect prefers-reduced-motion: reduce. Duration set to 0.01ms. No flashing, no disorientation for motion-sensitive users. globals.css → @media (prefers-reduced-motion)
✓
Screen reader text on action buttons — All row action buttons use entity-specific sr-only text: "Actions for {name}", not generic "Actions". Column headers stay generic. CLAUDE.md lesson → specific sr-only text
✓
Smooth transitions — 200ms sidebar collapse, 300ms fade-in/slide-in animations. Custom keyframes for accordion, fade, slide. All respect reduced motion preference. globals.css → @keyframes fade-in, slide-in-left
